01Shared architecture pattern
1
Polling, not webhookBoth bots use getUpdates on a 1-minute time-driven trigger. Apps Script Web App URLs always 302-redirect, which breaks Telegram's webhook delivery — polling sidesteps this entirely.
2
Nothing sends without a human tapEvery outbound message — public listing or private reminder — goes to Nick first as a review message with inline ✅ Approve / ❌ Reject buttons. Approve triggers the real send; Reject just clears the buttons.
3
Free translation via LanguageAppBoth reuse Apps Script's built-in LanguageApp.translate() — no paid API needed for JP/RU/DE/KM output.
4
Secrets live in Script Properties, never in chatBot tokens and chat IDs are added directly by Nick in each project's Script Properties. Each Apps Script project (Content Queue vs. PHC CRM) has its own independent property store — a token added to one is invisible to the other.

04Known gotchas — read before touching either bot
⚠
Two files are named PHC_AppsScript.gs A stale duplicate lives at ~/Desktop/phc-website/PHC Website (Claude Code)/PHC_AppsScript.gs — missing the referral columns and the entire Payment Reminder Bot. Always edit the one in ~/Desktop/Claude Code/phc-tools/. If a function "can't be found" right after pasting, this is the first thing to check.
⚠
Sheets silently converts typed chat IDs to Number — normalize before sending A chat/group ID typed directly into a Sheets cell comes back from getValues() as a JS Number, not a String. Passed straight into Telegram's API, this produces a 400 "chat not found" error even though the exact same value as a string works fine. Always run any Sheets-sourced chat ID through normalizeChatId(id) before using it — this bit the Payment Bot's groupChatId field and cost a long debugging session before the root cause was found. Apply this to any future feature reading a Telegram ID from a Sheet.
⚠
Sheets Date objects need explicit formatting A payDay cell comes back as a native JS Date; concatenating it raw into a message produces "Sat Jul 25 2026 00:00:00 GMT+0700...". Always format client-facing dates with Utilities.formatDate(d, Session.getScriptTimeZone(), 'MMMM d, yyyy') first.
⚠
A bot can only use ONE delivery mechanism at a time Webhook and getUpdates polling are mutually exclusive — if a webhook URL is ever set (even by accident, or a leftover from testing), polling silently returns empty forever regardless of real activity. testPaymentBotWebhookInfo() / the Content Bot's equivalent exist specifically to rule this out during debugging.
”
